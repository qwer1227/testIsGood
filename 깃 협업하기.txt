Plan A
1. 한 사람(임의로 A라 부른다)이 레파지토리를 만든다.
2. A가 레파지토리에 setting에 들어가서 사람들을 추가한다.(깃허브 아이디)
3. A가 레포지토리 주소를 다른 팀원들에게 공유한다.
	예) https://github.com/qwer1227/testIsGood.git
4. A가 공유한 원격저장소의 주소를 이용해 git bash에서 "git clone "원격저장소"를 입력한다
5. 4번의 과정을 거치면 git bash 현재 디렉토리위치상에 원격저장소에 있는 파일들이 모두 현재 위치에 저장이된다
6. 원격저장소에 있는 파일을 클론을 완료 했다면 폴더가 하나 생길것이고 그 폴더에 들어가서 작업을 하면 된다
7. 작업을 마쳤으면 git bash에서 git status를 입력한다 그러면 무조건 modified라는 문구가 빨갛게 나올것이며 이 상태에서 git add "원격저장소에 올릴 파일 이름"을 입력한다 다 올릴거면 git add .을 하면된다
8. add를 완료했으면 commit을 해야한다 commit 명령어는 git commit -m "남길문구"입니다.
9. commit을 완료했으면 push를 할 차례입니다 push단계는 실질적으로 원격저장소에 저장을 하는 단계입니다 명령어는 git push -u origin "브랜치이름"
